# CMS Content Model (Admin Panel)

## Core document types

1. `siteSettings`
   - site title
   - default SEO metadata
   - social links
   - contact details
2. `page`
   - title
   - slug
   - hero section
   - modular content blocks
   - SEO fields
3. `event`
   - event title
   - date/time
   - venue
   - ticket URL
   - price range
   - event description
   - featured image
   - structured data fields
4. `service`
   - service name
   - slug
   - target audience
   - content sections
   - CTA settings
   - SEO fields
5. `locationPage`
   - city/region
   - supported services
   - local proof content
   - FAQ items
   - SEO fields
6. `blogPost`
   - title
   - slug
   - author
   - publish date
   - body
   - category/tags
   - featured image
   - SEO fields
7. `testimonial`
   - name
   - organization
   - quote
   - rating
   - media asset
8. `faq`
   - question
   - answer
   - topic

## Reusable block types

- hero
- CTA strip
- testimonial carousel
- FAQ accordion
- feature grid
- video embed
- rich text block

## SEO field bundle (shared)

- meta title
- meta description
- canonical URL override
- noindex toggle
- OG image
- schema type selector

## Workflow roles

- Admin: full access
- Editor: content creation/editing/publish
- Marketing: SEO fields + page content updates
